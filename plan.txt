# Project Management Tool — AI Execution Plan

> **How to use this plan:** Execute one phase at a time. Each phase is fully self-contained — every feature introduced in a phase is also tested in that same phase. Do not start the next phase until `python manage.py check` (backend) or `npm run build` (frontend) passes clean.

---

## SECTION A — Workflow Orchestration (AI must follow these at all times)

These rules apply before, during, and after every phase. They are not optional.

### A.1 Plan Before You Build

- Enter plan mode for ANY non-trivial task (3+ steps or architectural decisions).
- If something breaks unexpectedly, STOP — do not keep pushing. Re-read the plan, identify what diverged, re-plan the corrected approach, then continue.
- Use plan mode for verification steps too, not just building. If a verification step fails, plan the fix before applying it.
- Write detailed specs before writing code. Ambiguity that surfaces during implementation should be resolved by updating the plan first.

### A.2 Subagent Strategy

- Use subagents to keep the main context window clean. Offload research, file exploration, and parallel analysis.
- One focused task per subagent. Do not give a subagent multiple unrelated jobs.
- For complex problems (e.g. "why is this query slow?"), spawn an Explore subagent rather than guessing.
- Never duplicate work a subagent is already doing. If you delegated a search, do not also run the same search yourself.

### A.3 Self-Improvement Loop

- After ANY correction from the user: immediately update `tasks/lessons.md` (create it if it doesn't exist) with the exact pattern that caused the mistake and the rule that prevents it next time.
- Format each lesson as: `**Mistake:** <what went wrong>` + `**Rule:** <what to do instead>`.
- At the start of each new session, read `tasks/lessons.md` first to avoid repeating known mistakes.
- Ruthlessly iterate — if the same mistake happens twice, the lesson was not specific enough. Rewrite it.

### A.4 Verification Before Done

- Never mark a task or phase complete without proving it works. Run the verification commands listed at the end of each phase.
- For backend changes: run the relevant test suite, hit the endpoint manually, confirm status code and response shape.
- For frontend changes: `npm run build` must pass (no TypeScript errors), tests must pass, and the feature must work in a real browser.
- Ask yourself before marking done: *Would a staff engineer accept this PR?* If not, fix it first.

### A.5 Demand Elegance

- For any non-trivial change: pause and ask "is there a simpler or more correct way?"
- If a fix feels like a workaround: stop, identify the root cause, implement the proper solution.
- Do not over-engineer simple things. A two-line fix does not need a utility function.
- Never silence a linter error with `# noqa` or `// eslint-disable` without a written comment explaining why.

### A.6 Autonomous Bug Fixing

- When given a bug: read the error, trace it to the source, fix it. Do not ask the user how to fix it.
- Use logs, failing tests, and stack traces as your primary inputs.
- Fix failing CI-style checks (type errors, test failures, lint errors) without being told to.
- Report what you found and what you changed — one clear summary after the fix, not a running commentary.

## SECTION B — Non-Negotiable Code Standards (enforced in every file)

### B.1 TypeScript — Frontend

- `"strict": true` in `tsconfig.json`. This must never be disabled.
- **Zero `any`** — if a type is unknown, use `unknown` and narrow it. If a third-party type is missing, write a local declaration file.
- Every function parameter and return type must be explicitly typed — no implicit inference for public API boundaries.
- Every API response must be typed against the interfaces in `src/types/`. Never destructure a raw axios response without casting it.
- Use `axios` for all HTTP calls. **Never use `fetch` directly.** All calls go through `src/api/client.ts`.
- No inline `useState<any>`. Every piece of state must have an explicit type.
- No `as unknown as T` casts unless there is a code comment explaining why it is safe.
- Props interfaces are defined above the component in the same file, named `<ComponentName>Props`.
- Enums for fixed string sets (status, priority, role) — define them in `src/types/` and import. Never use magic strings like `'in-progress'` scattered in components.

### B.2 React — Frontend

- **No `useEffect` to fetch data** — use React Query (`useQuery` / `useMutation`) exclusively for server state.
- **No prop drilling past 2 levels** — use Zustand store or React Query cache instead.
- Components are either UI (receives props, no API calls) or feature (uses hooks, no direct UI logic). Never mix both in one file above 100 lines.
- Custom hooks (`use*.ts`) own all business logic. Pages and components call hooks and render.
- **No direct `localStorage` access in components** — all token/auth storage goes through `authStore.ts`.
- Every `useMutation` has `onSuccess` (invalidate cache, show toast) and `onError` (show toast with message).
- Loading and error states are always handled — never render data without checking `isLoading` / `isError`.
- `React.memo` only when there is a measured performance problem. Do not pre-optimize.

### B.3 API Layer — Frontend

- Each resource has its own file in `src/api/` (`auth.ts`, `projects.ts`, `tasks.ts`, `comments.ts`).
- Every function in `src/api/` is typed: explicit parameter types and explicit return type `Promise<T>`.
- Extract the `data` from the envelope inside the API function, so callers receive the unwrapped payload directly.
- Handle 4xx errors by throwing an error with the message from `response.data.errors` so callers can display it.
- The axios client in `src/api/client.ts` handles: attaching `Authorization` header, retrying once on 401 with refresh token, redirecting to `/login` on refresh failure.
- Base URL comes from `import.meta.env.VITE_API_URL` only. Never hardcode `localhost:8000`.

### B.4 Django / Python — Backend

- **No untyped views** — every view function or ViewSet method has type annotations on `request` and return.
- **No raw SQL** — use Django ORM. If a complex query is needed, use `.annotate()` and `.aggregate()`.
- **No `except Exception`** — catch specific exceptions. If you must catch broadly, log the exception and re-raise or return a structured error.
- **Never expose internal errors to the client** — all unhandled exceptions are caught by the custom exception handler in `common/exceptions.py` which returns the envelope format.
- **No logic in serializer `create()` or `update()`** — inject `owner`, `created_by`, and `project` via `perform_create()` in the ViewSet.
- Every serializer field that is optional in input must have `required=False` and a sensible default.
- Model `save()` calls always pass `update_fields=[...]` when updating a subset of fields. Never call a bare `instance.save()` in a view.
- All imports are absolute, not relative. Use `from apps.users.models import User`, not `from ..users.models`.

### B.5 API Design — Backend

- **Every endpoint validates both request AND response.** Request: via DRF serializer. Response: always uses the envelope `{ "data": ..., "message": "...", "errors": null }` via the custom exception handler and explicit `Response()` calls.
- **Correct HTTP status codes — strictly enforced:**
  - `200` — successful GET or PATCH
  - `201` — successful POST (resource created)
  - `204` — successful DELETE (no body)
  - `400` — validation error (bad input)
  - `401` — missing or invalid token
  - `403` — authenticated but not permitted
  - `404` — resource not found (or intentionally hidden from unauthorized users)
  - `409` — conflict (duplicate unique constraint)
- **Never return 200 for an error.** Never return 500 in normal operation.
- All list endpoints are paginated — no endpoint ever returns an unbounded list.
- All filterable endpoints use `django-filter` `FilterSet` classes — never `request.query_params.get()` inline in a view.
- `get_queryset()` is always scoped to `request.user`. Never return objects the user cannot see, even with a 403 — return 404 to avoid leaking existence.
- Write serializers and read serializers are **separate classes**. Never use the same serializer for both input and output if the shapes differ.
- Passwords are **never** present in any serializer output. The `UserSerializer` used for responses must not include `password`.
- Soft-deleted records must **never** appear in any list or detail endpoint. Every `get_queryset` must filter `deleted_at__isnull=True` for soft-deleted models.

### B.6 Testing Standards

**Backend (Django `APITestCase`):**
- Every endpoint has tests for: success, missing required fields (400), wrong owner (404 not 403 for hidden resources), unauthenticated (401), invalid data types.
- `setUp` creates the minimum data needed — no test relies on seed data.
- Tests are deterministic — no `random`, no `time.sleep`, no external calls.
- Use `self.client.force_authenticate(user=self.user)` to authenticate, never manually set headers in tests.
- Assert both status code and response body shape for every test.

**Frontend (Vitest + Testing Library + MSW):**
- Every page component has tests.
- MSW intercepts all network calls — no real HTTP in tests.
- Test what the user sees, not implementation details — query by role, label, text. Never query by CSS class.
- Every form has tests for: empty submit (validation errors visible), valid submit (API called with correct payload), API error (error message shown).
- Loading states and empty states are tested.
- No `waitFor(() => expect(screen.getByText(...)))` with arbitrary timeouts — use `findBy*` queries which have built-in retry.

### B.7 Comment Style

Backend (Python):
```python
# store lowercase so comparisons don't fail on mixed-case input
email = email.lower()

# soft delete — callers filter on deleted_at__isnull so this row disappears from all lists
instance.deleted_at = timezone.now()
instance.save(update_fields=['deleted_at'])

# owner is implicitly a member, adding a duplicate row here would break the unique constraint
if role == 'owner':
    return
```

Frontend (TypeScript):
```ts
// token must exist before redirect or the next request will 401
setAuth(user, tokens)
navigate('/dashboard')

// keep filters in the URL so the view is shareable via link
setSearchParams({ status: value })

// close on mobile after navigation otherwise the overlay stays up
if (isMobile) setSidebarOpen(false)
```

Rules:
- One line. Explains WHY. The code already shows WHAT.
- No `// This function does X` above a function named `doX`.
- No block comments unless explaining a non-obvious algorithm.
- No TODO comments committed to main — either fix it now or create a tracked issue.

## SECTION C — Global Execution Rules

### C.1 Port Policy
- Backend: **8000** — set once, never changes
- Frontend: **3000** — `strictPort: true` in `vite.config.ts`
- Postgres: **5432** — set in `.env`, never hardcoded in settings

### C.2 Environment Variables
- Backend: all config via `python-decouple`. No `os.environ` directly. Every key has a `.env.example` entry.
- Frontend: only `VITE_` prefixed vars are exposed to the browser. Never put secrets in frontend env.
- `.env` is always gitignored. `.env.example` is always committed.

### C.3 Dependency Policy
- Never install a package to solve a problem that the already-installed stack can solve.
- Before adding a new package: check if Django ORM / DRF / React Query / Zustand / Tailwind already handles it.
- Every new package must be added to the appropriate `requirements/*.txt` or `package.json` immediately.

### C.4 API response envelope (all endpoints)

```json
{ "data": <payload or null>, "message": "string", "errors": null or { "field": ["msg"] } }
```

```python
from rest_framework.views import exception_handler
from rest_framework.response import Response


def custom_exception_handler(exc, context):
    response = exception_handler(exc, context)
    if response is not None:
        return Response(
            {'data': None, 'message': 'error', 'errors': response.data},
            status=response.status_code,
        )
    return response
```

### 2.2 apps/users/models.py

```python
import uuid
from django.contrib.auth.models import AbstractBaseUser, PermissionsMixin
from django.db import models
from .managers import UserManager


class User(AbstractBaseUser, PermissionsMixin):
    id = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)
    email = models.EmailField(unique=True)
    first_name = models.CharField(max_length=100)
    last_name = models.CharField(max_length=100)
    is_active = models.BooleanField(default=True)
    is_verified = models.BooleanField(default=False)
    is_staff = models.BooleanField(default=False)
    last_login_at = models.DateTimeField(null=True, blank=True)
    created_at = models.DateTimeField(auto_now_add=True)
    updated_at = models.DateTimeField(auto_now=True)

    USERNAME_FIELD = 'email'
    REQUIRED_FIELDS = ['first_name', 'last_name']

    objects = UserManager()

    class Meta:
        db_table = 'users'

    def __str__(self):
        return self.email

    @property
    def full_name(self):
        return f'{self.first_name} {self.last_name}'


class RefreshToken(models.Model):
    id = models.UUIDField(primary_key=True, default=uuid.uuid4, editable=False)
    user = models.ForeignKey(User, on_delete=models.CASCADE, related_name='refresh_tokens')
    token = models.CharField(max_length=500, unique=True)
    device_info = models.CharField(max_length=255, null=True, blank=True)
    is_revoked = models.BooleanField(default=False)
    expires_at = models.DateTimeField()
    created_at = models.DateTimeField(auto_now_add=True)
```

### 3.1 apps/users/serializers.py

Write these serializers:

**RegisterSerializer** — fields: `email`, `password`, `first_name`, `last_name`

- `validate()`: authenticate using `email.lower()` and `password`. If invalid, raise `ValidationError('Invalid credentials')`. If user `is_active=False`, raise `ValidationError('Account is disabled')`. Return user object on `validated_data['user']`.

**UserSerializer** — read-only. Fields: `id`, `email`, `first_name`, `last_name`, `is_verified`, `created_at`
- `full_name` via `SerializerMethodField`

**UpdateProfileSerializer** — fields: `first_name`, `last_name` only. Both optional.

### 3.2 apps/users/views.py

Write these views (use `APIView` not ViewSet for auth since they don't map to a model):

**RegisterView** (POST) — `permission_classes = [AllowAny]`
1. Run `RegisterSerializer(data=request.data)`
2. If invalid → 400
3. `user = serializer.save()`
4. Generate tokens using SimpleJWT's `RefreshToken.for_user(user)`
5. Return 201 with envelope: `{ "data": { "access": ..., "refresh": ..., "user": UserSerializer(user).data } }`

**LoginView** (POST) — `permission_classes = [AllowAny]`
1. Run `LoginSerializer(data=request.data)`
2. If invalid → 400
3. Get `user = serializer.validated_data['user']`
4. Update `user.last_login_at = timezone.now()`, save
5. Generate tokens
6. Return 200 with same envelope as register

**LogoutView** (POST) — `permission_classes = [IsAuthenticated]`
1. Get `refresh_token` from request body
2. Blacklist it using SimpleJWT: `token = RefreshToken(refresh_token); token.blacklist()`
3. Return 200 `{ "data": null, "message": "logged out", "errors": null }`

**TokenRefreshView** — re-export SimpleJWT's `TokenRefreshView` as-is, but wrap response in envelope using a subclass.

**MeView** (GET, PATCH) — `permission_classes = [IsAuthenticated]`
- GET: return `UserSerializer(request.user).data`
- PATCH: run `UpdateProfileSerializer`, save, return updated user

```python
from rest_framework.permissions import BasePermission
from .models import ProjectMember


class IsProjectMember(BasePermission):
    """User must be a member or owner of the project."""

    def has_object_permission(self, request, view, obj):
        return (
            obj.owner == request.user
            or obj.members.filter(user=request.user).exists()
        )


class IsProjectOwner(BasePermission):
    """Only the project owner can perform this action."""

    def has_object_permission(self, request, view, obj):
        return obj.owner == request.user


class IsProjectOwnerOrEditor(BasePermission):
    """Owner or editor role can modify the project."""

    def has_object_permission(self, request, view, obj):
        if obj.owner == request.user:
            return True
        return obj.members.filter(user=request.user, role__in=['owner', 'editor']).exists()
```

### 7.3 Date utilities — src/utils/date.ts

```ts
export function formatDate(iso: string | null | undefined): string {
  if (!iso) return '—'
  return new Date(iso).toLocaleDateString('en-US', { month: 'short', day: 'numeric', year: 'numeric' })
}

export function isOverdue(iso: string | null | undefined): boolean {
  if (!iso) return false
  return new Date(iso) < new Date()
}

export function relativeTime(iso: string): string {
  const diff = Date.now() - new Date(iso).getTime()
  const minutes = Math.floor(diff / 60000)
  if (minutes < 1) return 'just now'
  if (minutes < 60) return `${minutes}m ago`
  const hours = Math.floor(minutes / 60)
  if (hours < 24) return `${hours}h ago`
  return formatDate(iso)
}
```

### 7.4 API client — src/api/client.ts

```ts
import axios from 'axios'

const client = axios.create({
  baseURL: import.meta.env.VITE_API_URL,
  headers: { 'Content-Type': 'application/json' },
})

// attach access token on every request
client.interceptors.request.use((config) => {
  const token = localStorage.getItem('access_token')
  if (token) config.headers.Authorization = `Bearer ${token}`
  return config
})

// on 401, try to refresh — if that fails, send user to login
client.interceptors.response.use(
  (res) => res,
  async (error) => {
    const original = error.config
    if (error.response?.status === 401 && !original._retry) {
      original._retry = true
      const refresh = localStorage.getItem('refresh_token')
      if (!refresh) {
        window.location.href = '/login'
        return Promise.reject(error)
      }
      try {
        const { data } = await axios.post(`${import.meta.env.VITE_API_URL}/api/v1/auth/token/refresh/`, { refresh })
        localStorage.setItem('access_token', data.access)
        original.headers.Authorization = `Bearer ${data.access}`
        return client(original)
      } catch {
        localStorage.clear()
        window.location.href = '/login'
      }
    }
    return Promise.reject(error)
  }
)

export default client
```

### 7.6 Zustand stores

**src/store/authStore.ts**
```ts
import { create } from 'zustand'
import { persist } from 'zustand/middleware'
import type { User, AuthTokens } from '../types/auth'

interface AuthState {
  user: User | null
  isAuthenticated: boolean
  setAuth: (user: User, tokens: AuthTokens) => void
  clearAuth: () => void
}

export const useAuthStore = create<AuthState>()(
  persist(
    (set) => ({
      user: null,
      isAuthenticated: false,
      setAuth: (user, tokens) => {
        localStorage.setItem('access_token', tokens.access)
        localStorage.setItem('refresh_token', tokens.refresh)
        set({ user, isAuthenticated: true })
      },
      clearAuth: () => {
        localStorage.removeItem('access_token')
        localStorage.removeItem('refresh_token')
        set({ user: null, isAuthenticated: false })
      },
    }),
    { name: 'auth-store' }
  )
)
```

```tsx
  const isAuthenticated = useAuthStore((s) => s.isAuthenticated)
```

### 8.3 src/features/auth/useAuth.ts

```ts
import { useMutation } from '@tanstack/react-query'
import { useNavigate } from 'react-router-dom'
import toast from 'react-hot-toast'
import { login, register, logout } from '../../api/auth'
import { useAuthStore } from '../../store/authStore'

export function useLogin() {
  const setAuth = useAuthStore((s) => s.setAuth)
  const navigate = useNavigate()
  return useMutation({
    mutationFn: login,
    onSuccess: ({ data }) => {
      setAuth(data.user, { access: data.access, refresh: data.refresh })
      navigate('/dashboard')
    },
    onError: () => toast.error('Invalid email or password'),
  })
}

export function useRegister() {
  const setAuth = useAuthStore((s) => s.setAuth)
  const navigate = useNavigate()
  return useMutation({
    mutationFn: register,
    onSuccess: ({ data }) => {
      setAuth(data.user, { access: data.access, refresh: data.refresh })
      navigate('/dashboard')
    },
    onError: () => toast.error('Registration failed'),
  })
}

export function useLogout() {
  const clearAuth = useAuthStore((s) => s.clearAuth)
  const navigate = useNavigate()
  return () => {
    const refresh = localStorage.getItem('refresh_token')
    if (refresh) logout(refresh).catch(() => null)
    // clear local state regardless of server response
    clearAuth()
    navigate('/login')
  }
}
```
